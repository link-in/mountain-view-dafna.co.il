# 🚨 תיקון דחוף - בעיית קבצי JS

## הבעיה

השגיאה מראה שהדפדפן מנסה לטעון:
```
/next/static/chunks/80bd62c2443564c0.js
```

אבל צריך להיות:
```
/_next/static/chunks/80bd62c2443564c0.js
```

**חסר קו תחתון (`_`) בהתחלה!**

---

## 🔍 סיבות אפשריות

### 1. הקבצים לא הועלו נכון

**בדוק בשרת:**
- האם תיקיית `_next/static/` קיימת ב-`public_html/`?
- האם הקבצים בתיקייה?

**פתרון:**
- העלה מחדש את כל תיקיית `_next/` מתיקיית `out/`
- ודא שהמבנה נשמר: `public_html/_next/static/chunks/...`

### 2. ה-.htaccess מנתב גם קבצים קיימים

**פתרון:**
- עודכן ה-`.htaccess` - עכשיו הוא גם מטפל ב-`/next/` (בלי קו תחתון)
- העלה את ה-`.htaccess` החדש לשרת

### 3. הנתיב בשרת לא נכון

**בדוק:**
- האם הקבצים נמצאים ב-`public_html/_next/` או ב-`public_html/next/`?
- אם הם ב-`next/` (בלי קו תחתון), זה לא נכון!

---

## ✅ פתרון מיידי

### שלב 1: העלה את ה-.htaccess החדש

1. קח את הקובץ `.htaccess` מהשורש של הפרויקט
2. העלה אותו ל-`public_html/.htaccess` (החלף את הקיים)
3. ודא שהרשאות נכונות (644)

### שלב 2: ודא שהקבצים קיימים

**בדוק בשרת (דרך SSH או File Manager):**

```bash
# דרך SSH
ls -la /path/to/public_html/_next/static/chunks/
```

**אם התיקייה לא קיימת או ריקה:**
1. העלה מחדש את כל תיקיית `_next/` מתיקיית `out/`
2. ודא שהמבנה נשמר: `public_html/_next/static/chunks/...`

### שלב 3: בדוק את הנתיב

**פתח בדפדפן:**
```
https://mountain-view-dafna.co.il/_next/static/chunks/80bd62c2443564c0.js
```

**צריך לראות:**
- קוד JavaScript (לא HTML)
- Content-Type: `application/javascript`

**אם אתה רואה HTML:**
- הקובץ לא קיים בשרת
- העלה מחדש את תיקיית `_next/`

2. העלה מחדש את כל תיקיית `out/`
3. העלה את `.htaccess` החדש

```bash

tail -f /var/log/apache2/error.log
```
